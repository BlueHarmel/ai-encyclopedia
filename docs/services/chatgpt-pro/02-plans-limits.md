## 왜 이 챕터가 중요한가
요금/제한은 자주 바뀌기 때문에 숫자를 문서에 박아두면 빠르게 낡습니다. 이 문서는 **정답 숫자**보다 **확인 절차**를 표준화합니다.

## 확인 절차(권장)
1. 공식 가격/플랜 페이지 확인
2. 계정 내 사용량/한도 화면 확인
3. 조직 계약(있다면) 별도 문서 확인

## 운영 표준 테이블 (복붙용)

| 항목 | 값 | 확인 경로 | Last checked | 비고 |
|---|---|---|---|---|
| 플랜명 | 확인 필요 | 계정 설정 | YYYY-MM-DD | |
| 메시지/사용량 제한 | 확인 필요 | 사용량 화면 | YYYY-MM-DD | |
| 팀 기능 포함 여부 | 확인 필요 | 플랜 설명 | YYYY-MM-DD | |
| API 과금 연동 여부 | 확인 필요 | 조직 정책 | YYYY-MM-DD | |

## 비용 관리 루틴
- 주 1회: 사용량 스냅샷 기록
- 월 1회: 작업 유형별 사용량 회고
- 기능 릴리즈 후: 제한 정책 변경 여부 재확인

## 실전 프롬프트 (비용 민감 작업)
```text
아래 작업을 비용 민감 모드로 수행할 수 있게 프롬프트를 재작성해줘.
조건:
- 출력 길이 최소화
- 반드시 필요한 단계만 실행
- 중복 질문 방지
출력:
1) 최소 입력 템플릿
2) 실패 시 재시도 전략
3) 예상 토큰/질문 절감 포인트
```

## 확인 필요 / 직접 확인 필요
- 실제 가격 숫자
- 일/주/월 단위 정확 제한 값
- 조직 계약별 예외 정책
