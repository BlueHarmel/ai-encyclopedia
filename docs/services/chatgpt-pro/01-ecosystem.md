## 제공 표면(Surfaces)

| 표면 | 용도 | 개발자 추천 사용처 |
|---|---|---|
| Web | 대화형 분석/작성/파일 검토 | 요구사항 정리, 설계 리뷰, 긴 문서 초안 |
| Desktop/Mobile | 빠른 확인/메모/아이디어 캡처 | 이동 중 요약 확인, TODO 초안 |
| API(별도 제품) | 코드 기반 자동화 | CI 파이프라인, 내부 도구 연동 |

> 주의: UI 기능/버튼/도구 구성은 계정·플랜·시점에 따라 달라질 수 있음 (`확인 필요`).

## 계정/동기화 관점 체크
- 같은 계정으로 로그인 시 대화 이력/설정 동기화 가능(환경별 차이 `확인 필요`)
- 팀/조직 사용 시 권한·로그 정책은 조직 설정 우선

## 기본 진입점 추천
1. 설계/리뷰 중심 작업: Web 먼저
2. 자동화·반복 작업: API + 스크립트
3. 빠른 확인/요약: Mobile/Desktop

## 백엔드 개발자용 시작 프로필
- **Profile A (설계 중심)**: Web + 긴 컨텍스트 + 표 출력 강제
- **Profile B (구현 중심)**: 코드블록 출력 강제 + 테스트 항목 필수
- **Profile C (운영 중심)**: 장애 템플릿 + 롤백 체크리스트 고정

## 실전 프롬프트 (진입점 선택)
```text
내 작업 유형을 분류해줘.
입력: (문제 설명)
출력 형식:
1) 권장 진입점(Web/API/Desktop)
2) 예상 산출물
3) 검증 체크리스트
```

## 체크리스트
- [ ] 이번 작업이 대화형(Web)인지 자동화(API)인지 먼저 결정했는가
- [ ] 출력 형식을 먼저 고정했는가(표/코드/체크리스트)
- [ ] 민감정보 제거 후 입력했는가
