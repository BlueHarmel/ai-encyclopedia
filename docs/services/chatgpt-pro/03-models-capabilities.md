## 모델 선택 실전 규칙
- 빠른 반복/짧은 답: 속도 우선 모델
- 복잡한 설계/리뷰: 추론 우선 모델
- 코드 변경 제안: 코드 맥락 + 검증 단계 강제

## 지원 범위 표 (운영 템플릿)

| 항목 | 지원 | 메모 |
|---|---:|---|
| 텍스트 생성 | ✅ | 기본 기능 |
| 코드 생성/수정 제안 | ✅ | 검증 단계 필수 |
| 파일 입력 | 확인 필요 | UI/플랜에 따라 상이 |
| 웹 검색/브라우징 | 확인 필요 | 계정/시점별 상이 |
| 외부 연동 | 확인 필요 | 조직 정책 의존 |

## 실패 패턴과 회피법

### 패턴 A: 그럴듯한 단정
- 증상: 근거 없이 확정형 문장
- 회피 프롬프트:
```text
사실/해석/추정으로 분리해서 답해줘.
각 사실 문장에는 근거를 붙여줘.
근거 없으면 '확인 필요'로 표시해.
```

### 패턴 B: 코드 제안은 맞는데 테스트가 없음
- 회피 프롬프트:
```text
코드 제안 후 반드시 아래를 같이 출력해줘:
1) 테스트 케이스 3개
2) 린트/빌드 명령
3) 롤백 방법
```

### 패턴 C: 요구 범위 초과
- 회피 프롬프트:
```text
수정 파일은 아래 목록만 허용:
- src/auth.ts
- src/auth.test.ts
목록 외 파일 변경 금지.
```
