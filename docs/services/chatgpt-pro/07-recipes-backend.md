## Recipe A. 장애 대응 (Incident)
### 입력
- 에러 로그
- APM 메트릭
- 최근 배포 PR
### 출력
- 가설 목록
- 검증 순서
- 롤백 기준
### 검증 체크리스트
- [ ] 변경 파일 목록 파악
- [ ] 재현 테스트
- [ ] 롤백 전략 문서화

## Recipe B. API 설계/리뷰
### 입력
- 유스케이스
- 도메인 제약
### 출력
- OpenAPI 초안
- 에러 모델
- 버전 전략
### 검증 체크리스트
- [ ] 상태코드 일관성
- [ ] 에러 응답 스키마
- [ ] 하위호환성

## Recipe C. SQL 튜닝
### 입력
- SQL + EXPLAIN
- 인덱스 현황
### 출력
- 병목 원인
- 튜닝안
- 검증 절차
### 검증 체크리스트
- [ ] 실행계획 비교
- [ ] 응답시간 비교
- [ ] 쓰기성능 영향

### 입력
- 대상 파일
- 금지사항
### 출력
- 단계 분할 계획

### 검증 체크리스트
- [ ] 기능 동등성 테스트
- [ ] 린트/빌드
- [ ] 영향 범위 기록

## Recipe E. 보안 점검
### 입력
- 코드/설정

### 출력
- 취약점 후보
- 재현 경로
- 개선안
### 검증 체크리스트
- [ ] 인증/인가
- [ ] 입력 검증
- [ ] 민감정보 로그 노출

## 공통 프롬프트 골격
```text
역할: 시니어 백엔드 엔지니어
```
