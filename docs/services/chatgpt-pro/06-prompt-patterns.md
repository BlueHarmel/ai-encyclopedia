> Confidence: High

## 1) 요구사항 수집 템플릿
```text
너는 백엔드 아키텍트다.
아래 요청을 구현 가능한 요구사항으로 정리해줘.
출력:
1) 기능 요구사항
2) 비기능 요구사항(성능/보안/가용성)
3) 범위 밖 항목
4) 미정 의사결정 질문 목록
```

## 2) 설계 리뷰 템플릿
```text
다음 설계안을 리뷰해줘.
출력 형식:
- 장점 3
- 리스크 3
- 대안 2
- 권장안 1 (이유 포함)
```

## 3) 디버깅 템플릿
```text
증상, 로그, 최근 변경사항을 입력할게.
원인 가설을 우선순위로 정렬하고
각 가설별 검증 명령을 제시해줘.
```

## 4) 코드리뷰 템플릿
```text
아래 diff를 리뷰해줘.
관점: 버그/성능/보안/가독성/테스트
출력: 표 형태 + 바로 붙여넣는 리뷰 코멘트
```

## 5) 테스트 전략 템플릿
```text
기능 설명을 기반으로 테스트 전략을 설계해줘.
필수: 단위/통합/E2E 최소 케이스와 실패 케이스 포함.
```

## 6) 문서화 템플릿
```text
변경사항을 릴리즈 노트 형식으로 정리해줘.
필수: 사용자 영향, 마이그레이션 필요 여부, 롤백 방법
```

## 서비스 특화 튜닝 포인트
- 출력 길이 제한(장문 폭주 방지)
- 표 + 코드블록 강제(가독성 향상)
- 검증 단계 강제(테스트/롤백 누락 방지)
