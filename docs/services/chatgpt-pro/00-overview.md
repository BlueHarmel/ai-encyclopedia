# 00. Overview — ChatGPT Pro 101

> Updated: 2026-02-28  
> Plan used: Pro  
> Last checked: 2026-02-28  
> Confidence: Medium

## 3줄 요약
- ChatGPT Pro는 **요구사항 정리 → 설계 → 코드/문서 초안 → 검증 체크리스트 생성** 루프를 빠르게 반복할 때 가장 효율적입니다.
- 백엔드 개발에서는 장애 분석, API 설계 리뷰, 코드리뷰 코멘트 정리, 테스트 전략 초안화에 체감 효율이 큽니다.
- 강력한 출력 제어를 얻으려면 “역할/입력 제약/출력 포맷/검증 단계”를 프롬프트에 고정해야 합니다.

## 강한 작업 / 약한 작업

### 강한 작업
1. 장문 요구사항 정리 + 액션아이템 분해
2. 설계 리뷰(장단점/리스크/대안 비교)
3. 코드 변경 전 체크리스트 생성
4. PR 설명/릴리즈 노트/운영 문서 초안

### 약한 작업
1. 실시간 운영 상태 단정(외부 사실 최신성 필요)
2. 미확인 수치(비용/한도/정책) 단정
3. 맥락 없이 큰 코드베이스 전체 변경 지시

## 추천 사용 시나리오 Top 5
1. 장애 대응: 로그 기반 가설 목록 + 검증 순서 생성
2. API 설계: OpenAPI 초안 + 에러 모델 + 버전 전략
3. 코드 리뷰: diff 기반 위험도/테스트 보강 포인트 추출
4. 리팩터링: 단계 분할 + 롤백 포인트 설계
5. 문서화: 변경 요약/운영 공지/FAQ 자동 초안

## 금지/주의
- 고객 개인정보, 비밀키, 내부 비공개 원문 데이터는 입력 금지
- “확인하지 않은 가격/한도/정책”은 반드시 `확인 필요`로 표기
- 코드 변경 제안은 항상 테스트/린트/빌드/롤백 체크리스트를 함께 출력

## 개발자 기본 루프 (권장)
1. **질문**: 문제/목표/제약 정의
2. **계획**: 변경 단위 분할 + 검증 항목 생성
3. **실행**: 코드/문서 초안 생성
4. **검증**: 테스트/린트/빌드/리스크 확인
5. **기록**: PR 본문/회고/운영 문서화
