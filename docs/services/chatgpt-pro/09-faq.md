# 09. FAQ — ChatGPT Pro 101

> Updated: 2026-02-28  
> Last checked: 2026-02-28  
> Confidence: Medium

## Q1. 코드 수정을 그대로 적용해도 되나요?
아니요. 반드시 테스트/린트/빌드와 롤백 계획을 같이 확인해야 합니다.

## Q2. 긴 대화가 길어지면 품질이 떨어지나요?
떨어질 수 있습니다. 작업 단위를 스레드로 분리하세요.

## Q3. 가격/제한을 문서에 숫자로 써도 되나요?
변동 가능성이 커서 `Last checked`와 함께 기록하고, 불확실하면 `확인 필요`로 표시하세요.

## Q4. 로그를 그대로 넣어도 되나요?
민감정보 마스킹 후 넣는 것을 기본으로 하세요.

## Q5. 모델 선택은 어떻게 하나요?
속도 우선/추론 우선/코드 우선으로 작업 유형을 먼저 분류하세요.

## Q6. 환각이 의심될 때는?
사실/해석/추정 분리 출력을 강제하고 근거를 요구하세요.

## Q7. API 설계를 맡길 때 핵심은?
유스케이스/제약/에러 정책/버전 전략을 먼저 입력하세요.

## Q8. PR 리뷰 품질을 높이려면?
리뷰 관점(버그/성능/보안/가독성/테스트)을 프롬프트에 고정하세요.

## Q9. 어디까지 자동화해도 되나요?
운영 반영 전 단계(초안/분석/체크리스트)까지만 자동화하고 최종 반영은 사람 승인 권장.

## Q10. 문서 신뢰도를 높이는 방법은?
근거 링크 + Last checked + 불확실 정보 분리를 반드시 유지하세요.
