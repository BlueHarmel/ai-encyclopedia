## 표준 루프: 질문 → 계획 → 실행 → 검증

### Step 1) 질문(Problem framing)
```text
문제: 로그인 API에서 간헐적 500 발생
목표: 500 원인 가설 3개와 검증 순서 도출
제약: DB 스키마 변경 금지
```

### Step 2) 계획(Plan)
```text
출력 형식:
1) 가설 목록
2) 검증 순서
3) 실패 시 롤백 포인트
```

### Step 3) 실행(Execution draft)
- 코드/쿼리/설정 변경안
- 필요한 테스트 목록

### Step 4) 검증(Verification)
- 테스트/린트/빌드
- 성능 회귀 여부
- 장애 재현 여부

## 짧은 질의 vs 긴 질의

| 유형 | 기준 | 운영법 |
|---|---|---|
| 짧은 질의 | 한 번에 결정 가능 | 단일 질문 + 고정 출력 포맷 |
| 긴 질의 | 설계/비교/시나리오 다수 | 단계 분할 + 중간 검증 |

## 세션 쪼개기 기준
- 기능 단위 스레드 분리(인증/결제/검색)
- PR 단위 분리(브랜치와 1:1 매핑)
- 실험 스레드는 결과만 본문에 반영

## 실전 템플릿 (백엔드 작업)
```text
역할: 시니어 백엔드 리뷰어
입력: 요구사항, 현재 코드, 제약사항
출력:
- 변경안(코드블록)
- 테스트 명령
- 검증 체크리스트
- 롤백 계획
형식: Markdown 표 + 코드블록
```
