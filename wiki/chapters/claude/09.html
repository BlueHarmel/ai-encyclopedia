9. 고급 응용(자동화/통합)

응용 1) 생성→검증 2-pass 체인

요소내용
상황장문 정책 문서 배포 전 단정 문장 검토가 누락되어 정정이 반복된다.
입력원문 문서, 배포 대상 독자, 출력 형식(요약/액션/근거)
출력Pass1 초안 + Pass2 근거 점검 결과(유지/삭제/보류 태그)
검토포인트단정 문장마다 근거 링크가 연결되는지, 보류 항목이 분리되는지 확인
실패패턴Pass2 없이 초안을 바로 배포해 근거 누락 문장이 외부로 노출

응용 2) 작성자/검토자 역할 분리 자동화

요소내용
상황한 번의 프롬프트로 작성·검토를 동시에 처리해 논리 비약이 남는다.
입력작성자 프롬프트(초안), 검토자 프롬프트(근거 누락 탐지)
출력초안 문서 + "근거 없는 문장" 목록 + 수정 제안
검토포인트검토자 출력이 근거 링크 기준으로 문장을 지적하는지 확인
실패패턴검토자가 스타일 수정만 수행하고 사실 검증을 생략

응용 3) 채널별 출력 자동 변환

요소내용
상황같은 메시지를 메일/메신저/문서 채널마다 수동 재작성해 시간 소모가 크다.
입력핵심 메시지, 공식 근거 링크, 채널별 길이 제한
출력메일형(상세), 메신저형(5줄), 문서형(근거 포함) 3버전
검토포인트채널별 금지 포맷/길이 제한을 만족하는지 확인
실패패턴메신저용 결과가 장문으로 출력되어 전달 실패

Source: Claude Docs Overview, Claude API Overview, Anthropic News, Anthropic Release Notes(verified)
