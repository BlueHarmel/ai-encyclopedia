5. 핵심 기능별 설명(실사용 매뉴얼)

기능 A) 장문 문서 요약/재구성

항목운영 가이드
언제 쓰나회의록, 정책 초안, 이슈 정리처럼 길고 중복이 많은 문서를 압축할 때
입력 최소 세트원문 + 대상 독자 + 길이 제한 + 단정 금지
권장 출력 구조핵심 요약 / 결정사항 / 액션아이템 / 확인 필요
실패 패턴독자·길이·금지사항을 빼고 요청해 장황하거나 모호한 결과가 나옴
실제 사용 예시 1

상황: 30~40페이지 회의록을 당일 임원 보고로 압축해야 함.

실행: "임원용 8줄 + 실행항목 5개 + 확인 필요 2개" 형식으로 요청.

결과 활용: 보고용 1페이지와 실무 액션리스트를 분리 배포.

기능 B) 초안 품질 점검(논리 비약/중복/불명확 표현)

항목운영 가이드
언제 쓰나이미 작성된 초안이 있는데 품질 편차를 줄여야 할 때
입력 최소 세트초안 본문 + 점검 관점(논리/중복/표현) + 수정 강도(최소/중간)
권장 출력 구조문제 문장 번호 / 문제 유형 / 수정 제안 / 근거 확인 필요 여부
실패 패턴사실 검증 대상 문장을 "표현 개선" 단계에서 확정 문장으로 바꿔버림
실제 사용 예시 2

상황: 기획안 검토 회의에서 "핵심이 안 보인다"는 피드백 반복.

실행: "논리 비약 문장만 번호로 뽑고, 최소 수정안만 제시" 요청.

결과 활용: 리뷰 시간이 줄고, 수정 범위를 합의하기 쉬워짐.

기능 C) FAQ 초안 생성(배포 전 검증 분리형)

항목운영 가이드
언제 쓰나정책/공지 배포 직전, 예상 질문 대응을 빠르게 준비할 때
입력 최소 세트정책 원문 + 예상 질문 범위 + 답변 톤 + 근거 미확인 분리 규칙
권장 출력 구조FAQ 본문 + 확인 불가/직접 확인 필요 목록
실패 패턴요금/플랜/제한치 같은 변동 항목을 검증 없이 확정 답변으로 작성
실제 사용 예시 3

상황: 정책 변경 직후 고객 문의량 급증.

실행: 질문 10개를 먼저 생성하고, 변동 가능 문장은 자동으로 "직접 확인 필요" 태그 부여.

결과 활용: 상담팀이 안전한 문장만 즉시 사용하고, 불확실 답변은 검증 큐로 보냄.

운영 체크포인트(공통)

  • 결과를 초안으로 취급하고 사람 검토를 통과한 뒤 배포한다.
  • 근거가 불명확한 문장은 무조건 "확인 불가/직접 확인 필요"로 분리한다.
  • 플랜/가격/조직 권한처럼 변동값이 큰 항목은 문서에 숫자 단정으로 남기지 않는다.
