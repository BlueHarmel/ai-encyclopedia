7. 기초 예제(따라하기)

공통 실행 규칙 (세 예제 모두 동일)

  • 출력 구조를 요약/근거/확인 필요 3단으로 고정한다.
  • 근거가 없는 단정 문장은 최종본에서 제외한다.
  • 배포 전 사람 검토에서 용어·정책 표현을 마지막으로 점검한다.

예제 1) 장문 회의록을 임원 보고용 1페이지로 압축

요소내용
상황회의록이 길어 핵심 의사결정 전달이 늦어진다.
입력회의록 원문 + 대상 독자(임원) + 길이 제한(8줄) + 추측 금지 규칙
출력핵심 3개 / 결정 2개 / 확인 필요 2개
검토포인트결정사항과 미확정 항목이 섞이지 않았는지 확인
실패패턴미확정 내용을 확정 문장으로 옮겨 후속 커뮤니케이션 충돌 발생
요청 예시
- 목표: 임원 보고용 1페이지 요약
- 독자: 경영진
- 형식: 핵심 3 / 결정 2 / 확인 필요 2
- 제약: 근거 없는 확정 문장 금지

예제 2) 정책 문서를 고객 FAQ 초안으로 변환

요소내용
상황정책 변경 직후 반복 문의가 증가한다.
입력정책 본문 + 예상 질문 10개 + 답변 톤(중립/간결) + 근거 미확인 분리 규칙
출력FAQ 10세트 + 확인 불가/직접 확인 필요 목록
검토포인트요금/플랜/제한치가 단정 답변으로 들어가지 않았는지 확인
실패패턴근거 없는 확정 답변 배포 후 정정 공지 발생
요청 예시
- 목표: 고객 FAQ 10개 작성
- 형식: Q/A 각 2~3문장
- 필수: 불확실 항목은 "직접 확인 필요"로 별도 표기
- 금지: 정책/가격 수치 추정

예제 3) 이슈 로그를 "실행 계획" 문서로 재구성

요소내용
상황여러 채널에 흩어진 이슈 로그를 실행 가능한 작업으로 정리해야 한다.
입력이슈 로그 묶음 + 우선순위 기준 + 출력 템플릿(담당/기한/선행조건)
출력실행 항목 목록 + 의존성 + 확인 필요 리스크
검토포인트담당자 공란, 기한 미정 항목이 별도 표시되는지 확인
실패패턴우선순위 기준이 없어 긴급도 낮은 항목이 앞에 배치됨
요청 예시
- 목표: 이번 주 실행계획으로 변환
- 형식: [우선순위][담당][기한][선행조건][리스크]
- 제약: 담당/기한 미정은 "확인 필요"로 분리
