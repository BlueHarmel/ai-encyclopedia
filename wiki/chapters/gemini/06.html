Gemini

6. 인터페이스별 사용 흐름(Web/앱/API/CLI 해당 시)

Web/앱 흐름

  • 주제 입력 후 요약/근거/확인 필요 3분할 요청
  • 출력 길이/독자 수준 지정으로 결과 일관성 유지
  • 최종 공유 전 확인 필요 문장만 재검토
예시 블록 1

업데이트 공지를 만들 때 "핵심 3개 + 영향 2개 + 확인 필요 2개"로 고정하면 실무 공유 속도가 빨라진다.

API 흐름

  • 입력: 질문/자료/출력 포맷 분리
  • 출력: 스키마(JSON/섹션) 고정으로 후속 시스템 연동
  • 검증: 근거 링크 누락 시 재요청
예시 블록 2

문서 자동화에서는 "생성 → 검증 → 배포본 변환" 3단계로 나누면 오류 확산을 줄일 수 있다.

CLI 관련 주의

  • sources.json 범위에서 범용 Gemini CLI를 단정할 공식 근거는 제한적이다.
  • 근거 부족 항목은 확인 불가/직접 확인 필요로 분리한다.

확인 불가/직접 확인 필요: 계정/플랜별 기능 제공 차이, 지역별 정책 차이.
