5. 핵심 기능별 설명(입력/출력/옵션/실패 패턴)

기능 A) 링크 기반 리서치 요약

  • 입력: 주제, 참고 링크 목록, 출력 형식(요약/근거/확인 필요)
  • 출력: 핵심 요약 + 링크 근거 + 미확인 항목 분리
  • 옵션: 길이 제한, 표/불릿 형식, 공식 도메인 우선 정렬
  • 실패 패턴: 링크 없는 주장, 공식성 낮은 출처 혼입
예시 블록 1 — 경쟁 제품 조사

비교 문장마다 근거 링크 1개 이상을 강제하면, 후속 검토에서 "검증 불가 주장"을 빠르게 걸러낼 수 있다.

예시 블록 2 — 주간 동향 리포트

"핵심 5개 + 근거 링크 + 확인 필요" 템플릿을 고정하면 작성자 변경 시에도 문서 품질 편차가 줄어든다.

기능 B) 비교표 생성

  • 입력: 비교 대상 2~5개, 판단 기준(기능/운영/리스크)
  • 출력: 기준별 비교표 + 항목별 근거 링크
  • 옵션: 중요도 정렬, 누락 항목 표시, 불확실 항목 태깅
  • 실패 패턴: 마케팅 문구 중심 서술, 필수 기준 누락
예시 블록 3 — 도입 의사결정 표

필수 기준을 기능/운영/제약 3축으로 먼저 고정하면, 비교표가 단순 소개가 아닌 의사결정 자료로 바로 사용된다.

예시 블록 4 — 누락 탐지

근거가 없는 셀을 "직접 확인 필요"로 표시하면, 확정 내용과 검증 대기 항목을 명확히 분리할 수 있다.

기능 C) 검증 큐 분리

  • 입력: 초안 결과 + 검증 규칙(공식 Source 우선)
  • 출력: 반영 가능/직접 확인 필요/보류 항목 분류
  • 옵션: 우선순위 태깅(상/중/하), 담당자 할당용 주석
  • 실패 패턴: 확인 필요 항목을 확정 사실로 혼합

Source: Genspark 공식 사이트(접근 가능 시), 공식 GitHub 저장소(sparkle/omaha), verified 문서 기준
