Genspark

4. 핵심 기능 사용법 (입력/출력/제약/실패 패턴)

  • 입력: 주제 + 비교기준 + 길이 + 링크의무
  • 출력: 요약/근거 링크/확인 필요 분리
  • 제약: 미검증 단정 금지, 공식 사이트 403 시 직접 확인 필요 표기
  • 실패 패턴: 링크 없는 요약, 공식성 낮은 출처 혼입, 검증시각 누락, 확인 필요 항목의 확정 반영
  • API/기능 개요 커버리지: 공식 사이트(직접 확인 필요 가능) + sparkle/omaha로 기능 흐름 보완
  • 운영 게이트: 반영 가능/보류/직접 확인 필요 3분류로 승인 흐름 유지
