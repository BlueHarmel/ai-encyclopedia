4. 핵심 기능 사용법 (입력/출력/제약/실패 패턴)

4-1) 기본 작업 흐름

  • 입력: 작업 목표, 변경 범위, 성공 조건
  • 출력: 코드 수정안 + 검증 절차 + 롤백 포인트
  • 제약: 미검증 사실 단정 금지, 위험 명령 자동 실행 금지
  • 실패 패턴: 성공 조건 없이 대규모 변경 시도

4-2) 업데이트 반영

  • 입력: changelog 기간/버전
  • 출력: 영향 범위 요약 + 적용 체크리스트
  • 제약: changelog 근거 없는 기능 단정 금지
  • 실패 패턴: 알파 릴리즈를 안정판처럼 가정
