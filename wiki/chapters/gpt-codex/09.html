9. 고급 응용(자동화/통합)

응용 1) 생성→검증 2-pass 파이프라인

요소내용
상황코드 생성 속도는 빠르지만 검증 누락으로 재작업이 반복된다.
입력작업 목표, 변경 허용 범위, 실패 재현 조건
출력Pass1 최소 수정 코드 + Pass2 테스트 보강/검증 리포트
검토포인트Pass2에서 실패 시나리오가 실제 재현 가능한지 확인
실패패턴Pass1 결과를 바로 병합해 운영 버그가 재발

응용 2) changelog 연동 자동 업데이트

요소내용
상황changelog 확인은 하지만 실제 코드 반영 우선순위가 정리되지 않는다.
입력공식 changelog 링크, 현재 버전, 영향 가능 모듈 목록
출력영향 파일 목록 + 적용/보류 분류 + 재검토 날짜
검토포인트분류 근거가 공식 링크와 1:1로 연결되는지 확인
실패패턴비공식 요약을 근거로 사용해 잘못된 반영 결정

응용 3) PR/릴리즈 문서 동시 생성

요소내용
상황개발자용 PR 설명과 운영자용 릴리즈 노트를 이중 작성한다.
입력코드 diff, 테스트 결과, 배포 영향 범위
출력PR 본문 + 운영 릴리즈 노트 + 롤백 가이드
검토포인트두 문서의 버전/영향 범위/검증 결과가 일치하는지 확인
실패패턴PR과 릴리즈 노트 간 정보 불일치로 배포 혼선 발생

Source: Codex Overview, Codex Changelog, openai/codex Repository/Releases
