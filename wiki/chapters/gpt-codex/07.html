7. 기초 예제(따라하기)

실행 전 체크리스트

  • 변경 허용 파일/폴더와 금지 경로를 먼저 명시했는가?
  • 성공 조건(테스트 통과, 동작 동일성)을 문장으로 고정했는가?
  • 출력 요구 형식을 "패치 + 테스트 + 변경 요약"으로 지정했는가?

예제 1) 로그인 예외 처리 버그 수정

요소내용
상황운영 로그에서 로그인 API가 간헐적으로 500을 반환한다.
입력오류 로그 20줄, 재현 단계, 변경 허용 파일(service/auth.ts)
출력최소 수정 패치 + 정상/예외 테스트 2건 + 변경 이유 3줄
검토포인트패치가 허용 파일 밖을 건드리지 않았는지, 테스트가 실패 재현을 포함하는지
실패패턴재현 테스트 없이 방어 코드만 추가해 "고쳐진 것처럼 보이는" 상태

예제 2) 금액 계산 함수 테스트 3종 생성

요소내용
상황할인 계산 함수에서 경계값 처리 누락 가능성이 있다.
입력함수 시그니처, 정상/예외/경계 조건, 반올림 규칙
출력테스트 3종(정상/예외/경계) + 실패 시 원인 후보 목록
검토포인트경계값(0, 최대값, 음수) 커버 여부와 예상 에러 메시지 일치 여부
실패패턴정상 케이스만 생성되어 운영 결함 재발

절차 요약: 재현 → 최소 수정 → 테스트 보강 → 변경 요약 순서를 고정하면 재작업 빈도를 줄일 수 있다.
