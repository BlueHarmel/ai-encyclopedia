11. 비용/플랜(검증 가능 범위 + 확인 필요 분리)

확인 가능한 범위 vs 직접 확인 필요

구분문서화 원칙확인 경로
확인 가능한 범위 기능/변경 이력은 Codex Overview/Changelog 및 GitHub Releases 기준으로 추적한다. developers.openai.com/codex, developers.openai.com/codex/changelog, github.com/openai/codex/releases
직접 확인 필요 실시간 과금 단가, 사용량 한도, 조직 계약 조건은 환경별로 달라 단정하지 않는다. 조직 관리자 콘솔, 계약 문서, 공식 가격 안내 페이지

도입 의사결정 체크리스트(숫자 단정 대신 측정 절차)

  • 파일럿 기간을 2~4주로 고정했는가?
  • 측정 지표를 PR 준비시간, 테스트 누락률, 리뷰 반려율로 통일했는가?
  • 파괴적 변경은 dry-run/승인 게이트를 통과하도록 분리했는가?
  • changelog 반영 항목을 즉시/파일럿/보류로 분류했는가?
  • 월 단위 회고 문서에 근거 링크와 재검토 날짜를 남겼는가?
예시 블록 1 — PR 준비시간 절감률 계산

도입 전후 2주 평균 PR 준비시간을 비교해 (전-후)/전×100으로 산출한다. 산출값은 changelog 반영 시점과 함께 기록해 원인 추적이 가능하도록 유지한다.

예시 블록 2 — 테스트 누락률 기준 운영

주간 샘플 PR 20건에서 정상/예외/경계 테스트 포함 여부를 점검한다. 누락률이 높으면 프롬프트 템플릿에 테스트 3종 요구를 고정한다.

예시 블록 3 — 비용 문서 작성 규칙

내부 위키에는 금액 숫자를 고정하지 않고 확인 날짜, 확인자, 공식 링크만 기록한다. 분기마다 재확인해 최신 상태를 유지한다.

확인 불가/직접 확인 필요

  • OAuth/API 과금 세부 단가
  • 시간/토큰 단위 사용 한도 및 초과 정책
  • 조직/플랜별 기능 차등 범위
