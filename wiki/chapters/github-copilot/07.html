GitHub Copilot

7. 기초 예제(따라하기)

실행 전 체크리스트

  • 변경 허용 파일/금지 경로를 명시했는가? (예: service/ 허용, migrations/ 금지)
  • 성공 조건을 테스트 기준으로 적었는가? (정상/예외/경계)
  • 출력 형식을 코드 + 테스트 + 변경 요약으로 고정했는가?

예제 1) 함수 구현 초안 만들기

요소내용
상황신규 할인 계산 함수를 빠르게 추가해야 한다.
입력함수 목적, 입출력 타입, 예외 조건, 변경 허용 파일
출력최소 구현 코드 + 단위 테스트 초안 + 변경 이유 3줄
검토포인트예외 처리(음수/상한 초과) 포함 여부, 타입 오류 발생 여부
실패패턴목적만 전달해 null/경계 처리 누락

예제 2) 테스트 3종(정상/예외/경계) 보강

요소내용
상황운영 버그 재발로 테스트 커버리지 보강이 필요하다.
입력대상 함수, 실패 로그, 재현 단계
출력정상 1/예외 1/경계 1 테스트 + 실패 원인 후보
검토포인트재현 테스트가 먼저 실패하는지, 패치 후 통과하는지 확인
실패패턴정상 케이스만 생성해 운영 결함 재발

예제 3) PR 설명 초안 자동 생성

요소내용
상황변경 이유 누락으로 리뷰 왕복이 길어진다.
입력변경 파일 목록, 테스트 결과, 위험 구간
출력PR 본문(배경/변경/검증/리스크) + 리뷰 질문 목록
검토포인트리스크 섹션 누락 여부, 롤백 경로 기재 여부
실패패턴자동 생성 본문을 검토 없이 최종본으로 사용

Source: GitHub Copilot Docs, Copilot Quickstart, GitHub Changelog(Copilot)
