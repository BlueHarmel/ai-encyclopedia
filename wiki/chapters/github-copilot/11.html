직접 확인 필요 실시간 가격 숫자, 조직 계약 조건, 정책 토글 기본값은 환경마다 달라 단정하지 않는다. 조직 관리자 콘솔/계약 문서/공식 가격 안내 페이지

도입 의사결정 체크리스트(숫자 단정 대신 측정 절차)

  • 측정 기간을 2~4주 파일럿으로 고정했는가?
  • 지표를 PR 준비 시간, 테스트 누락률, 리뷰 반려율로 통일했는가?
  • preview 기능은 별도 저장소에서만 실험하도록 분리했는가?
  • 작은 PR(기능 단위 분할) 기준을 적용했는가?
  • 주 1회 회고에서 즉시/파일럿/보류 분류를 갱신하는가?
구체 예시 1 — PR 준비시간 절감률 계산

도입 전/후 2주 평균 PR 준비시간을 비교해 (도입 전 - 도입 후) / 도입 전 × 100으로 산출한다. 수치와 함께 changelog 반영 시점을 회고 문서에 기록한다.

구체 예시 2 — 테스트 누락률 측정

PR 20건 샘플에서 정상/예외/경계 테스트 포함 여부를 점검한다. 누락률이 높으면 팀 공통 프롬프트에 테스트 3종 요구를 강제한다.

구체 예시 3 — 가격 문서 갱신 규칙

내부 위키에는 가격 숫자를 고정하지 않고 확인 날짜, 확인자, 공식 링크만 기록한다. 분기마다 재확인해 업데이트한다.

확인 불가/직접 확인 필요

  • 플랜별 최신 가격 숫자(통화/세금/계약 조건 포함)
  • 조직별 사용량 한도, 초과 정책, 개별 계약 조항
  • 특정 IDE·조직 정책 조합에서의 세부 기능 노출 범위
